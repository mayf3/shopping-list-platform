# 购物清单管理平台 - 详细需求文档

> 本文档作为 Codex 开发的需求输入，请基于此文档进行开发。

## 1. 项目概述

### 1.1 目标
构建一个轻量级的 Web 购物清单管理平台，支持个人及家庭使用，部署在阿里云 ECS（2核2G/3M带宽/40G盘）上。

### 1.2 目标用户
- 用户本人（博士生，技术背景）
- 家庭成员（非技术用户）

### 1.3 核心价值
- 可视化购物清单管理
- 预算跟踪与消费统计
- 多端访问（手机/电脑浏览器）
- 简单易用，非技术用户也能操作

---

## 2. 功能需求

### 2.1 购物清单管理（核心）

#### 2.1.1 物品 CRUD
- ✅ 添加物品：名称、数量、单位、分类、优先级（高/中/低）、预估价格、备注、商店、购买链接
- ✅ 编辑物品：修改任何字段
- ✅ 删除物品：支持确认删除
- ✅ 标记已购买：一键标记，记录实际购买价格
- ✅ 批量操作：多选标记已购买、多选删除

#### 2.1.2 清单视图
- **列表视图**：按分类分组显示，每组可折叠
- **卡片视图**：物品以卡片形式展示，带图片
- **商店视图**：按商店分组，方便线下采购
- **优先级视图**：高优先级置顶

#### 2.1.3 分类系统
预设分类：
- 📦 食品生鲜（蔬菜水果、肉禽蛋奶、粮油调味、零食饮料）
- 📦 日用品（个人护理、家居清洁、纸品）
- 📦 电子产品（数码配件、家用电器）
- 📦 服装鞋帽
- 📦 家居用品
- 📦 图书文具
- 📦 儿童用品（玩具、桌游、绘本）
- 📦 云服务（服务器、域名、SSL证书）
- 📦 其他

支持自定义分类和子分类。

#### 2.1.4 搜索与筛选
- 关键词搜索
- 按分类筛选
- 按状态筛选（待购买/已购买/已取消）
- 按优先级筛选
- 按商店筛选

### 2.2 预算管理

#### 2.2.1 预算设定
- 月度总预算
- 分类预算（食品、日用、电子等）
- 单次购物预算

#### 2.2.2 花费跟踪
- 记录实际购买价格
- 自动计算已花费/剩余预算
- 超预算预警（红色提示）

#### 2.2.3 消费统计
- 月度消费汇总
- 分类消费饼图
- 月度消费趋势折线图
- 最常购买物品 Top 10

### 2.3 购买历史

#### 2.3.1 历史记录
- 所有已购物品记录（含购买日期、价格、商店）
- 按时间线查看
- 按分类查看

#### 2.3.2 智能提醒
- 常购物品周期识别（如：牛奶大约每 2 周买一次）
- 临期提醒（根据上次购买时间推断）
- 定期购物提醒

### 2.4 用户与权限

#### 2.4.1 简单认证
- 用户名/密码登录（轻量级，不需要复杂 OAuth）
- 至少支持 2 个用户（本人 + 家人）
- 可以不做注册，直接后台配置账号

#### 2.4.2 权限
- 管理员：所有操作
- 普通用户：添加/编辑/查看，不能删除预算配置

### 2.5 数据管理

#### 2.5.1 导入导出
- 导出购物清单为 JSON/CSV
- 导入购物清单（从 JSON）
- 支持从现有飞书清单数据迁移

#### 2.5.2 数据备份
- 自动每日备份（SQLite 文件备份）
- 手动备份/恢复

---

## 3. 非功能需求

### 3.1 性能要求
- 页面首屏加载 < 3 秒
- 支持 2-5 人同时使用
- 适配 3M 带宽（前端资源需压缩优化）

### 3.2 部署要求
- Docker 容器化部署
- 支持 HTTPS（Let's Encrypt 免费证书）
- 占用内存 < 512MB（给其他服务留空间）
- 磁盘占用 < 1GB

### 3.3 兼容性
- 浏览器：Chrome、Safari、Firefox（最近 2 个版本）
- 移动端：iOS Safari、Android Chrome
- 响应式设计，手机优先

### 3.4 安全
- HTTPS 强制
- 简单但够用的认证
- CSRF 防护
- 输入校验

## 4. 技术建议

### 4.1 推荐技术栈
考虑到服务器配置（2核2G），推荐轻量级方案：

**方案 A（推荐）：全栈 JavaScript**
- 前端：React + Vite + Tailwind CSS
- 后端：Hono / Fastify（轻量）
- 数据库：SQLite（通过 better-sqlite3）
- 部署：Docker 单容器或 docker-compose

**方案 B：Python**
- 前端：React + Vite
- 后端：FastAPI + SQLAlchemy
- 数据库：SQLite
- 部署：Docker Compose（前端 + 后端 2 容器）

### 4.2 数据库 Schema 建议

**核心表：**
- `items`：物品（id, name, quantity, unit, category, priority, status, estimated_price, actual_price, store, note, url, created_at, purchased_at）
- `categories`：分类（id, name, icon, parent_id, sort_order）
- `budgets`：预算（id, category_id, amount, period, month）
- `purchase_history`：购买历史（id, item_id, price, store, purchased_at）
- `users`：用户（id, username, password_hash, role）

### 4.3 API 设计
RESTful 风格：
- `GET /api/items` - 获取清单
- `POST /api/items` - 添加物品
- `PUT /api/items/:id` - 更新物品
- `DELETE /api/items/:id` - 删除物品
- `PATCH /api/items/:id/purchase` - 标记已购买
- `GET /api/stats/summary` - 消费统计
- `GET /api/budgets` - 预算管理
- `GET /api/history` - 购买历史

## 5. 页面设计建议

### 5.1 页面列表
1. **首页/仪表盘**：本月预算概览、待购买数量、最近购买记录
2. **购物清单页**：主页面，物品列表，支持多种视图切换
3. **添加/编辑物品**：弹窗或独立页面
4. **购买历史页**：时间线 + 统计图表
5. **预算管理页**：设置预算、查看消费图表
6. **设置页**：用户管理、分类管理、数据导入导出

### 5.2 移动端适配
- 底部 Tab 导航（清单 / 历史 / 预算 / 设置）
- 添加物品的浮动按钮（FAB）
- 左滑删除、右滑标记已购买
